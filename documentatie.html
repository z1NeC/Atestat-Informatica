1. Argument

Alegerea temei proiectului de atestat, „Wave — Platformă Digitală Premium de Închirieri Auto”, a fost motivată de dorința de a explora și implementa cele mai noi tehnologii web într-un context practic și utilitar. Într-o eră digitală în continuă expansiune, industria de rent-a-car necesită soluții software care să ofere nu doar funcționalitate, ci și o experiență de utilizare (UX) de înalt nivel.

Prin acest proiect, am urmărit crearea unui ecosistem digital complet care să simuleze fluxul real al unui business de închirieri: de la captarea atenției prin design, până la gestionarea datelor și procesarea unei rezervări. Am pus accent pe performanță, interactivitate și estetică modernă.

Principalele obiective tehnice ale proiectului au fost:

  • Implementarea unui sistem de generare dinamică a conținutului pentru a reduce timpul de încărcare și a ușura mentenanța.
  • Integrarea unor servicii externe (API-uri) pentru a oferi o reprezentare vizuală realistă a flotei.
  • Crearea unui sistem de teme (Light/Dark) bazat pe preferințele utilizatorului, utilizând variabile CSS native.
  • Dezvoltarea unui formular complex, validat prin JavaScript, care să asigure colectarea corectă a datelor clienților.
Pagina 3

2. Descrierea Tehnologiilor Utilizate

Proiectul a fost dezvoltat utilizând tehnologiile fundamentale ale web-ului modern, fără dependențe externe masive, pentru a demonstra măiestria în utilizarea limbajelor de bază.

2.1 HTML5 & Structură Semantică

Utilizarea HTML5 a permis crearea unei structuri bine definite, accesibile și optimizate SEO. Am utilizat tag-uri semantice precum <nav>, <header>, <section>, <footer> și <article> pentru a organiza logic informația.

Exemplu de structură semantică pentru cardul auto:
<article class="fleet-card"> <div class="card-image"></div> <div class="card-content"> <h3>Marca Model</h3> <div class="specs">...</div> <div class="pricing">...</div> </div> </article>

De asemenea, am integrat SVG-uri direct în cod pentru a asigura o claritate maximă a iconițelor și a elementelor grafice (cum ar fi loader-ul animat sau formele decorative din fundal) indiferent de rezoluția ecranului.

Pagina 4

2.2 CSS3 & Design Sistem

Designul vizual al platformei Wave se bazează pe conceptul de Glassmorphism și Modern Minimalist. Am creat un sistem de design scalabil utilizând variabile CSS (Custom Properties).

Definirea paletei de culori și a variabilelor de stare:
:root { --primary: #2563eb; --navy-900: #0f172a; --white: #ffffff; --radius-lg: 20px; --transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1); } body.dark-mode { --white: #0a1628; --navy-900: #ffffff; --gray-50: #112240; }

Pentru layout am utilizat intensiv Flexbox și CSS Grid, asigurând o adaptabilitate completă pe orice dispozitiv. Animațiile au fost realizate utilizând @keyframes și proprietatea transition, oferind o interfață „vie” care reacționează la interacțiunile utilizatorului.

2.3 JavaScript & Dinamică

Limbajul JavaScript a fost coloana vertebrală a interactivității. Am optat pentru un stil de programare modular, separând responsabilitățile: gestionarea bazei de date de mașini, logica de filtrare, managementul temei și validarea formularelor.

Funcții Cheie Implementate:

  • Intersection Observer: Pentru a declanșa animațiile de fade-in și slide-up pe măsură ce utilizatorul face scroll.
  • LocalStorage: Pentru a persista setările de Dark Mode și datele din comenzi între reîncărcările paginii.
  • Asynchronous Requests: Utilizate pentru a prelua imagini dinamice prin Imagin Studio API.
  • DOM Manipulation: Generarea a peste 50 de carduri auto dintr-o singură sursă de date JSON.
Logica de observare a elementelor pentru animație:
const observer = new IntersectionObserver((entries) => { entries.forEach(entry => { if (entry.isIntersecting) { entry.target.classList.add('animated'); observer.unobserve(entry.target); } }); }, { threshold: 0.1 }); document.querySelectorAll('[data-animate]').forEach(el => observer.observe(el));

3.1 Sistemul de Navigare și Preloader

Experiența utilizatorului începe cu un Preloader sofisticat, realizat în SVG, care elimină senzația de „flash” vizual până când toate resursele sunt încărcate.

3.1.1 Meniul Hamburger (Mobile)

3.2 Secțiunea Hero și Particule

Secțiunea Hero este elementul principal de atracție visuală. Aceasta combină animații de text cu un sistem de particule plutitoare generate prin JavaScript.

Sistemul de Particule: Am creat o funcție care generează 30 de elemente de tip <div> cu poziții, dimensiuni și viteze de animație aleatorii. Acestea adaugă profunzime vizuală fără a afecta performanța, spre deosebire de un fișier video de mari dimensiuni.

Funcția de creare a particulelor:
function createParticles() { const container = document.getElementById("particles"); for (let i = 0; i < 30; i++) { const p = document.createElement("div"); p.classList.add("particle"); p.style.left = Math.random() * 100 + "%"; p.style.top = Math.random() * 100 + "%"; p.style.animationDelay = Math.random() * 5 + "s"; container.appendChild(p); } }

Object.entries(carDatabase).forEach(([make, data]) => { Object.entries(data.models).forEach(([model, info]) => { const card = document.createElement("div"); card.className = `fleet-card ${bodyTypeMap[info.caroserii[0]]}`; card.innerHTML = ` <div class="card-img"> <img src="${getImageUrl(make, model)}"> </div> <div class="card-info"> <h3>${make} ${model}</h3> <p>${info.pret}€ / zi</p> </div> `; grid.appendChild(card); }); });

3.4 Formularul de Rezervare Multi-Step

Pentru a oferi o experiență cât mai plăcută, am divizat procesul de rezervare în 3 pași clari. Acest lucru reduce încărcarea cognitivă a utilizatorului și crește rata de finalizare a formularelor.

Funcționalități pas-cu-pas:

  1. Pasul 1 (Mașina): Dropdown-uri interconectate. Selectarea mărcii actualizează automat modelele, anii și culorile disponibile folosind JavaScript.
  2. Pasul 2 (Perioada): Calcularea automată a duratei între data de preluare și data de returnare.
  3. Pasul 3 (Date Personale): Validarea prin Regex (expresii regulate) a CNP-ului, numărului de telefon și adresei de email.

Validarea datelor: Nu permitem trecerea la următorul pas dacă input-urile obligatorii sunt goale sau incorect formatate. Am creat o funcție centralizată validateStep(n) care verifică toate câmpurile pasului curent.

Validarea CNP-ului și a Email-ului:
const patterns = { email: /^[^\s@]+@[^\s@]+\.[^\s@]+$/, tel: /^\d{10}$/, cnp: /^\d{13}$/ }; function validateField(id, pattern) { const el = document.getElementById(id); if (!pattern.test(el.value)) { showError(id, "Format incorect"); return false; } return true; }

După finalizarea cu succes, datele sunt colectate într-un obiect orderData, primesc un ID unic de tipul CMD-XXXXXX și sunt salvate în localStorage, simulând trimiterea către un server real.

3.5 Managementul Temelor (Dark Mode)

Implementarea Dark Mode nu este doar un moft estetic, ci o necesitate pentru confortul vizual al utilizatorului. Sistemul meu schimbă întregul design folosind o singură clasă pe tag-ul <body>.


LIGHT MODE

DARK MODE

Persistența Temei:

Pentru a evita „efectul de bliț” (site-ul să pornească pe alb și să sară pe negru după încărcare), am plasat un script mic în <head> care verifică localStorage imediat ce pagina începe să se încarce.

Script de inițializare a temei (plasat în Head):
(function() { const theme = localStorage.getItem('wave_theme'); if (theme === 'dark') { document.documentElement.classList.add('dark-mode'); } })();

4. Integrare API și Stocare

Un punct forte al proiectului este utilizarea Imagin Studio API. Acest serviciu extern îmi permite să afișez imagini de înaltă calitate pentru orice mașină, fără a stoca mii de fișiere local.

Construirea URL-ului pentru API-ul de imagini:
function getImageUrl(make, model, color) { const url = new URL("https://cdn.imagin.studio/getimage"); url.searchParams.append("customer", "hrjavascript-mastery"); url.searchParams.append("make", make); url.searchParams.append("modelFamily", model); url.searchParams.append("zoomType", "fullscreen"); if (color) url.searchParams.append("paintId", colorMap[color]); return url.toString(); }

Configurarea Datelor: Rezervările sunt stocate în localStorage sub cheia wave_rent_orders. Am implementat și un fișier comenzi.json care servește drept model pentru structura datelor, pregătind aplicația pentru o eventuală integrare cu un backend real (Node.js/PHP).

6. Concluzii

Realizarea proiectului Wave mi-a permis să pun în practică și să consolidez cunoștințele de informatică dobândite pe parcursul liceului la Colegiul Național „A.T. Laurian”. Am demonstrat că prin utilizarea corectă a limbajelor de bază (HTML, CSS, JS) se pot crea platforme complexe, estetice și perfect funcționale, fără a depinde de framework-uri externe greoaie.

Provocarea cea mai mare a fost gestionarea stării formularului multi-step și asigurarea unei interfețe fluide în momentul schimbării temelor. Rezolvarea acestor probleme m-a ajutat să înțeleg mai bine conceptele de Event Handling, State Management și DOM Manipulation.

Consider că platforma dezvoltată reprezintă un punct de pornire solid pentru o aplicație comercială, fiind pregătită pentru scalare și integrare cu sisteme de baze de date SQL sau platforme de plată online.
